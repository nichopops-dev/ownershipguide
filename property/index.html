How to use this section

Most users do better when they move in this order: framework → calculator → mechanics. If you start with deep mechanics pages without knowing whether the broad decision makes sense, it is easy to optimise the wrong thing.

  • Use core ownership pages when you are still deciding whether a property move is rational at all.
  • Use calculators when you need quick scenario testing with your own numbers.
  • Use mechanics pages when you already know the move might make sense and want to identify cash traps, timing drag, and exit friction.

How we build this page

This hub is organised around decision intent, not just topic labels. The goal is to help visitors find the right path quickly, especially when the real problem is not “What is property?” but “Can I buy?”, “Can I finance this safely?”, or “Will my sell-then-buy move break if timing slips?”

  • Core model first: we try to show total exposure before optimisation.
  • Decision routes second: we group tools and guides by real questions users are asking.
  • Transition friction included: the site now covers pricing, timeline, bridging, OTP, commission, and extension-of-stay because those are often where plans fail in practice.
